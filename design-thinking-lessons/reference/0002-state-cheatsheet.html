State Cheat Sheet

Terms

Duplicated state
The same fact tracked independently in more than one place. Nothing forces the copies to agree, so they eventually diverge.
Lifting state up
Moving state from a component to its closest common ancestor with the other components that need it, then passing it down as props.
Derived state
A value computed from other state at render time rather than stored in its own useState. A useEffect whose only job is syncing two state values is a sign one of them should be derived instead.
Global / shared client state
State read by components with no reasonably close common ancestor — reached via Context or a library (Zustand, etc.), not the default choice.

Decision order

AskIf yes →
Can I compute this from other state/props right now?Derived state — don't store it
Does only one component (or its children) need it?Local state
Do siblings need the same answer?Lift to closest common ancestor
Is the common ancestor unreasonably far up the tree?Global/shared client state
Is this actually a fact that lives server-side?Reconsider whether it should be client state at all (→ Lesson 3)
