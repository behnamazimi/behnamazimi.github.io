Data Fetching Cheat Sheet

Terms

Data waterfall
Sequential fetches where a later request's start doesn't actually depend on an earlier one's result — just written after an await.
Parallel fetching
Starting independent requests before awaiting any of them (call first, await Promise.all([...]) together).
Request memoization
Identical fetch calls within one render pass are automatically deduped by Next.js. React.cache extends this to non-fetch data sources.

The one question that separates waterfall from necessary sequence

Does the later call's arguments depend on the earlier call's result? If no — parallelize with Promise.all. If yes — it's genuinely sequential; wrap it in <Suspense> so it doesn't block everything else.
