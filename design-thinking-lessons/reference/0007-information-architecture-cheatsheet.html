Information Architecture Cheat Sheet

Terms

Normalization
Structuring data so each fact lives in exactly one place, referenced elsewhere by key rather than copied.
Denormalization
Deliberately storing a derived or duplicated value for read performance, accepting a staleness window as the cost.
Staleness
The gap between when source data changes and when a cached/denormalized copy catches up.

The judgment call this lesson tests

Not everything that looks like duplication is duplication. If two values represent different facts that currently happen to be equal (like an order line's price-at-purchase vs. today's live price), storing both is correct — they will diverge on purpose over time, and that divergence is the point.
