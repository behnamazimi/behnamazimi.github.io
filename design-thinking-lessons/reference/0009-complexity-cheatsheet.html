Complexity Cheat Sheet

Terms

Essential complexity
Inherent to the problem itself. No architecture choice removes it, only manages it.
Accidental complexity
Introduced by tools, technique, or habit rather than required by the problem. Unlike essential complexity, it can genuinely be eliminated.
Premature optimization
Spending effort optimizing before measurement shows it's actually the bottleneck. Not an argument against optimizing — an argument against guessing which 3% matters.

Quick test

Ask: if you rewrote this feature from scratch with perfect foresight, would this cost still exist? If yes — essential. If no — accidental, and worth fixing once you've measured it's part of the real 3%.

Primary sources
