Code Review Cheat Sheet

The purpose

Improve overall code health over time. Not perfection — approve once a change definitely improves it.

What to check, in order

Functionality
Does it do what it's meant to, and is that good for the people using it?
Complexity
Could this be simpler? Essential to the problem (Lesson 9), or accidental?
Tests
Correct, well-designed, and pitched at the right Testing Trophy layer (Lesson 11).
Naming
Do names say what things are?

How to say it

  • Comment on the code, never the person.
  • Prefix optional polish with Nit:.
  • Unresolved after a few rounds → escalate (call, then tech lead/maintainer), don't loop forever in comments.
