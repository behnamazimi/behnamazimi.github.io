Design Docs Cheat Sheet

ADR vs. design doc

ADRDesign doc
One decisionOne feature/system, possibly several decisions
Context / Decision / ConsequencesProblem / Goals & Non-goals / Proposed design / Alternatives considered / Risks
Quick, often written after decidingWritten before building, for readers who weren't in the room

The two sections that do the real work

  • Alternatives considered — proves tradeoff analysis happened, not just "first thing that worked."
  • Risks — state the failure mode before it happens. If Lesson 12's incident had a line here, it stops being a surprise.

Primary source

adr.github.io (ADR half; design-doc structure above is practical convention, not one canonical source)
